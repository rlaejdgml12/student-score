## Git·GitHub 기초
- `git add`: 변경된 파일을 스테이징 영역(커밋 대기열)에 올림. 아직 저장소에 기록된 건 아니고, "이번 커밋에 포함시킬 파일"을 고르는 단계.
- `git commit -m "메시지"`: 스테이징된 변경사항을 로컬 저장소에 기록(확정). `-m`은 message의 약자로, 별도 에디터 없이 커밋 메시지를 바로 입력하게 해주는 옵션.
- `git push`: 로컬 저장소에 쌓인 커밋들을 GitHub(원격 저장소)로 업로드. 새 저장소에서 처음 push할 땐 `git push -u origin main`처럼 `-u`(`--set-upstream`)로 로컬 브랜치와 원격 브랜치를 연결해줘야, 다음부터 `git push`만 쳐도 어디로 보낼지 자동으로 기억함.
- 처음 겪은 문제: 새 GitHub 저장소는 기본 브랜치가 `main`인데, 로컬 `git init`은 기본적으로 `master` 브랜치로 시작돼서 push 시 `src refspec main does not match any` 에러 발생 → `git branch -M main`으로 로컬 브랜치 이름을 `main`으로 바꿔서 해결.

## 클래스와 데이터
클래스는 붕어빵을 찍어내는 **빵틀**, 객체(인스턴스)는 그 틀로 찍어낸 **붕어빵 하나하나**. 클래스 자체는 설계도일 뿐 메모리를 차지하지 않고, `new`로 만들어진 객체마다 각자의 데이터(필드 값)를 따로 가짐.

## 기본형 vs 참조형
- 기본형 변수(`int`, `double`, `boolean` 등): 변수에 **값 자체**가 저장됨. 다른 변수에 대입하면 값이 복사되므로, 이후 한쪽을 바꿔도 다른 쪽엔 영향 없음.
- 참조형 변수(`String`, 객체 등): 변수에는 값이 아니라 **객체가 있는 메모리 주소**가 저장됨. 다른 변수에 대입하면 주소가 복사되므로, 두 변수가 **같은 객체를 가리키게 됨**. 그래서 한쪽 변수로 객체 내부 값을 바꾸면, 같은 주소를 가리키는 다른 쪽 변수로 봐도 그 변경이 그대로 보임.
